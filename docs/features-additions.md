# 追加機能 設計ドキュメント — GEOSPATIAL OPS

「いかにも高度な作業をしているように見える」演出を強化するための追加機能の設計をまとめる。実装対象は `index.html` のメインスクリプト（単一ファイル構成は維持）。

- 編集の前提・座標系の約束事は [../CLAUDE.md](../CLAUDE.md) を必ず参照すること。
- 本書は「何を・どこに・どう動かすか」を定義する設計メモであり、動作ガイドではない。

## 確定した方針（grilling の結果）

| # | 論点 | 決定 |
|---|---|---|
| 1 | 既存と重複する機能 | **既存は強化、重複する新規は作らない** |
| 2 | ターミナル風コンソールの配置 | **左カラムに 4 枚目のパネルとして追加** |
| 3 | 壁紙用途での負荷対策 | **負荷キャップ・トグルは設けず見た目最優先** |
| 4 | geo オーバーレイのリアルさ | **実在都市を結ぶ＋空間固定の疑似太陽で「それっぽく」** |
| 5 | 起動/アラート/キー演出のトリガー | **自動中心。キー演出はフォーカス時のみ動くボーナス** |
| 6 | ドキュメント形式 | 本ファイル（`docs/features-additions.md`）に新規作成 |

## 効果音は対象外

WebAudio による効果音は今回のスコープから除外する（ユーザー指示）。

## A. 既存機能の強化（4 件）

重複する新規パネルは作らず、既存実装を拡張する。

### A1. DOWNLINK SPECTRUM にオシロ波形モードを追加
- 対象: `spDraw()` / `#spectrum`。
- 内容: 既存の FFT バーに加え、数秒ごとに**オシロスコープ波形**（合成サイン＋ノイズの折れ線）モードへ切り替える。`spMode` で交互表示。
- 調整: 波形色は既存トークン系（cyan/violet/amber）に合わせハードコード。

### A2. 解析タスク群（proc）の動的化
- 対象: `P[]` / `tickProc()` / `#proc`。
- 内容: 固定 3 本（TLE/FRAME/GEO）に加え、完了時にラベルを**ランダムなタスク名へ差し替え**て湧き直す（例: `KEY BRUTEFORCE` / `PACKET REASSEMBLY` / `ORBIT SOLVE`）。本数は据え置き（レイアウト不変）。

### A3. 世界時計の追加（TOKYO / ZULU）
- 対象: topbar の `.stat` 群（`#utc` / `#met` 隣）。
- 内容: `UTC` と `MISSION ELAPSED` に加え、**TOKYO（JST）** と **ZULU（=UTC 表記）** を追記。`tickClocks()` で毎秒更新。

### A4. EVENT STREAM ログの完全生成式化
- 対象: `MSGS` / `logLine()` / `#log`。
- 内容: 固定文配列から、**IP・ポート・ハッシュ・CVE 番号・座標を都度ランダム生成**する方式へ。テンプレート関数群（`genIP()` / `genHex(n)` / `genCVE()`）を追加し、文面の重複感を消す。

## B. geo オーバーレイ（新規 5 件 / `#globe` 上、`draw()` 内）

すべて既存の `ll2v → spin → project`、表裏判定 `depth>=0` に従う。実在都市テーブルを共有する。

```
CITIES = [大阪, 東京, ソウル, シンガポール, シドニー,
          ニューヨーク, ロンドン, モスクワ, サンパウロ, ケープタウン, ...]
```

### B1. 攻撃トレース弧 `ATTACK ARC`（`drawArcs`）
- 2 都市間を**大圏（great-circle）**で結ぶ弧を球面補間で描画。
- 弧上を光のパルスが端から端へ走る。一定間隔で新しいペアを生成、寿命つきでフェードアウト。
- 同時本数に上限（例: 5）。前面は明るく、奥は薄く。

### B2. ターゲットロック HUD `TARGET LOCK`（`drawLocks`）
- 前面半球のランダム点に**照準リング＋十字＋ LAT/LON 読み**を表示。
- 数秒で次のターゲットへ遷移。`LOCKING → LOCKED` のラベル遷移。

### B3. 昼夜ターミネーター `DAY/NIGHT`（`drawTerminator`）
- **空間に固定（ゆっくりドリフト）した疑似太陽方向**を基準に、夜側半球を暗く陰影づけ。
- 地球は従来どおり高速自転し、その下を明暗境界が通過して見える（物理的に自然）。

### B4. レーダースイープ `RADAR SWEEP`（`drawRadar`）
- 地球の手前で回転する**スキャンライン**と、検出ブリップ（短命の光点）。
- 同心リングはごく薄く。

### B5. 都市ヒート点 `CITY HEAT`（`drawCityHeat`）
- 実在都市に**脈動するグロー点**。前面のときだけ描画。OSAKA は既存マーカーと色を分ける。

## C. 衛星系（新規 3 件）

### C1. 地上フットプリント円 `GROUND FOOTPRINT`（`drawFootprint`）
- 各衛星の直下点に通信圏を表す**円を球面に投影**。前面のみ。

### C2. AOS/LOS イベント `AOS/LOS`
- 衛星が前面に入る/出る瞬間を検出し、EVENT STREAM へ `SAT-0x ACQUIRED` / `LOST` を流す。
- 状態は各 `sat` に `wasFront` を保持して立ち上がり/立ち下がりを判定。

### C3. 衛星リンク線 `SAT LINK`（`drawSatLinks`）
- 前面の衛星同士、または衛星↔地上局（OSAKA）を結ぶ**明滅リンク**。メッシュ風。

## D. パネル・全画面演出（新規 4 件）

### D1. ターミナル風コンソール `CONSOLE`（新パネル）
- 配置: **左カラムに 4 枚目のパネル**（SIGINT / TELEMETRY / EVENT の下）。flex 配分を調整。
- 内容: `nmap` / `traceroute` 風の出力が**行ごとに自動タイプ**されていく擬似コンソール。`#console` に追記、行数上限でスクロール。
- 駆動: `tickConsole()`（既存ループのスロットルに乗せる）。

### D2. 起動シーケンス `BOOT SEQUENCE`（全画面オーバーレイ）
- 読み込み直後、`BOOTING… / LINK ESTABLISHED / ACCESS GRANTED` 等を数秒流してから UI を表出。
- `#boot` を最前面に置き、完了で `opacity` フェードアウト→`display:none`。壁紙のリロードでも毎回再生。

### D3. キー操作演出 `KEY FX`（ボーナス）
- 任意キー押下で `ACCESS GRANTED` 等のフラッシュ。`keydown` リスナー追加。
- 壁紙（キー不可）では発火しないが無害。

### D4. アラート状態 `ALERT STATE`（全画面ティント）
- 数十秒ごとに自動発火。画面を一瞬赤くし `⚠ INTRUSION DETECTED` を明滅 → 自動復帰。
- `#alert` オーバーレイ＋ CSS アニメ。EVENT STREAM にも対応ログ。

## 実装上の約束

- **単一ファイル維持**。`index.html` のメインスクリプト・`<style>`・`.stage`/左カラム HTML に追記する。
- **座標系は既存規約厳守**（`sx=CX-X`、`depth>=0` 前面、正射投影）。詳細は [../CLAUDE.md](../CLAUDE.md)。
- **配色**: UI は `:root` トークン、キャンバスは既存同様ハードコード（両者を揃える）。
- **描画順**: `draw()` 内で「ターミネーター（陰）→ 既存地球/海岸線 → ヒート点 → 弧 → 衛星/フットプリント/リンク → ロック → レーダー」の順を基本とし、奥行きの破綻がないよう調整。
- **負荷キャップなし**（決定 3）。ただし弧・ブリップ等は寿命管理で配列が無限増殖しないようにする（メモリリーク防止であって描画間引きではない）。
