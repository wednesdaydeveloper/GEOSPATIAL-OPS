# CLAUDE.md — 「GEOSPATIAL OPS」管制室風ダッシュボード

このリポジトリは、**いかにも高度な作業をしているように見える**フルスクリーンの管制室／ハッカー風ダッシュボードです。中心に自転する地球儀（実データの海岸線・大陸塗り）、周回する人工衛星、各種テレメトリパネル、暗号解読風の演出が動きます。実際のネットワーク通信や解読は一切しておらず、すべて見た目だけの演出です。

## 動かし方

- 依存ライブラリ・ビルド・通信は**一切不要**。`index.html` をブラウザでダブルクリックして開くだけ。オフラインで動作します。
- 全画面表示（Mac は `⌃⌘F`）にすると効果的。
- 海岸線データと陸地マスクは Base64 等でファイルに**埋め込み済み**。

## ファイル構成

単一ファイル `index.html` のみ。中身は大きく4ブロック:

1. `<style>` — CSS。配色は `:root` のカラートークンで一元管理。
2. `<script>` その1 — `const COAST=[...]`(陸地リングの座標) を埋め込んだデータ専用スクリプト。
3. `<script>` その2(メイン) 冒頭 — `const MASK_B64="..."`(陸地ビットマップ) を埋め込み。
4. `<script>` その2 残り — 描画・アニメーションの全ロジック。

> 編集対象はほぼ常に `index.html` のメインスクリプト。`COAST` と `MASK_B64` は生成済みデータなので通常は触りません（再生成方法は末尾参照）。

## 座標系と投影の約束事（重要）

ここを誤ると地図が反転・崩壊します。過去に何度も踏んだ落とし穴なので必ず守ってください。

- **`ll2v(lat,lon)`**: 緯度経度→単位ベクトル `[cos(lat)cos(lon), sin(lat), cos(lat)sin(lon)]`。
- **`spin(p, a)`**: 地球点を Y軸（極軸）まわりに角度 `a` だけ回し、半径 `R` を掛ける。地球の自転＝この `a`(=`earthAngle`)。
- **`project(x,y,z)`**: 正射投影（perspective なし）。
  - 画面X = **`CX - X`**（`CX + X` ではない）。これが**東西反転を防ぐ要**。視線が -Z 方向なので +X は画面左に対応するため、符号を反転している。
  - `ROLL` は地軸の傾き。**現在 `ROLL=0`（地軸は垂直）**。値を変えると画面内ロール（地軸の左右の傾き）になる。`ROLL` はピッチ（見下ろし角）ではない点に注意。
  - `depth = z`（カメラ深度）。**前面半球 = `z >= 0`**。海岸線・経線・衛星・大阪などの表裏判定はすべて `q.depth >= 0` で行う。
  - `scale = 0.65 + 0.35*clamp(z/R)`（マーカーの遠近サイズ用）。
- **`tilt3` / `projTilt`**: 地軸 `drawAxis` 用の補助。`ROLL` ロールを適用した3D座標と、その正射投影。
- 透視投影は使っていない（塗りと線がズレないよう正射で統一した経緯がある）。

## 主要モジュールと調整ポイント

### 地球儀
- **大陸の塗り `fillGlobe()`**: ベクタ多角形ではなく、**陸地ビットマップ `MASK_B64`(480×240) を球面に逆投影**してピクセル単位で塗る方式。`FILL_FS`(=192) のオフスクリプトキャンバスに描いて `drawImage` で拡大。`isLand(lon,lat)` がマスク参照。
  - 巨大リング（ユーラシア等）でベクタ塗りが破綻したため、この方式に変更した経緯がある。**ベクタ塗りに戻さないこと。**
  - 陸の色は `fillGlobe` 内の `d[p]=74; d[p+1]=156; d[p+2]=134; d[p+3]=108;`（RGBA）で調整。
- **海岸線**: `draw()` 内、`COASTV`(=COAST を単位ベクトル化)を `spin`→`project` して前面/背面を描く。前面は明るく、背面は薄く。
- **経線・赤道 `graticule`**: `meridians`(経線30°毎) + 赤道のみ。`draw()` 内 `dl()` の `globalAlpha`/`strokeStyle` で濃さ調整。
- **地軸 `drawAxis()`**: 北極〜南極を貫く線。`ROLL` に追従。極に N/S ラベル。
- **大阪マーカー**: `OSAKA=ll2v(34.69,135.50)`。脈動リング＋十字＋ラベル。前面のときだけ描画。色は金系（`#ffcf5a`/`rgba(255,207,90,...)`）。

### 人工衛星
- **`sats`**: 6基。各 `{rmul, inc, node, phase, speed, trail, col}`。色は `PALETTE`。
- **軌道半径は `rmul`(倍率)で保持し、`satPos()` 内で `R*s.rmul` と描画時に計算**する。
  - 過去のバグ: 初期化時に `r=R*…` で固定したが、その時点で `R=0`(resize前)のため全衛星が中心に重なり消えていた。**半径は必ず描画時に `R` を掛けること。**
- **`drawSat()`**: 本体＋ソーラーパネルのグリフ。動径方向に姿勢を合わせる。
- 軌道リング・トレイルは `draw()` 内。ドラッグ(`dragYaw`)で軌道も一緒に回るよう `rotY(satPos(...), dragYaw)` を通す。自転 `earthSpin` は地球のみ、`simTime` は衛星の周回位相を駆動（ドラッグ中も周回継続）。

### 操作（ドラッグ）
- 横ドラッグのみ有効（縦ドラッグは無効化済み）。`dragMove` は **`dragYaw -= dx*0.006`**（符号はマイナス。`sx=CX-X` の反転に合わせて手の動きと一致させている）。
- ドラッグ中は自動回転停止、離して約2.5秒後に再開（`idleT`）。

### ハッカー風演出
- **コードレイン `rainDraw()`**: 地球の背後の `#rain` キャンバス。濃さは CSS `#rain{opacity:...}` と、`rainDraw` 内のフェード `rgba(5,8,13,0.10)`・文字色・`RFS`(文字サイズ)・スロットル `rainAcc<0.045` で調整。
- **16進パケット `tickHex()`**: 左上 `#hexpkt`。傍受風の16進ダンプ。
- **デクリプト(文字スクランブル) `tickDecrypt()`**: 右上 `.decrypt`。`DC_TARGETS` の文字列を左から順に確定。確定部=シアン、未確定=スクランブル(`DC_SCR`)。`KEY 0x…`＋進捗バー。`accdc>0.07` で更新。
- **再構築球 `reconDraw()`**: 右カラム「SIGNAL RECON」パネルの `#recon`。緯度経度クアッド `QUADS`(15×24) を `reconOrder`(シャッフル順) に従って順次表示。**穴だらけ→完成**を演出し、100%で `✓ MESH RECONSTRUCTED`→崩して再構築をループ。
  - 速度: `reconP += dt*0.17`。完成保持: `reconHold=1.2`。最前線の発光面: `fresh` 判定。細かさ: `latN/lonN`。
- **グリッチ**: タイトル `<h1 class="glitch" data-text="...">`。CSS の `@keyframes gl1/gl2` で RGBずれ。
- **ASCIIアート**: 左カラム「SIGINT FEED」パネルの `<pre class="ascii">`（衛星のアスキー絵）。`.ascii` に明滅アニメ。

### UIパネル
- レイアウトは CSS グリッド: 上バー / `.main`(左カラム・中央ステージ・右カラム) / 下ティッカー。
- 左: SIGINT(ASCII) / SYSTEM TELEMETRY(ゲージ) / EVENT STREAM(ログ)。
- 右: CONSTELLATION(衛星ロスター) / SIGNAL RECON(再構築球) / DOWNLINK SPECTRUM(スペクトラム＋進捗バー)。
- ログ文言は `MSGS`、ティッカーは `buildTicker()`、スペクトラムは `spDraw()`。

### 配色
- `:root` のカラートークン（`--cyan`/`--amber`/`--violet`/`--gold`/`--green`/`--blue`/`--text`/`--muted` 等）で UI 全体を制御。
- キャンバス描画はトークンを参照できないため**色がハードコード**されている（`draw()`/`fillGlobe()`/`drawSat()`/`reconDraw()` 等）。配色を大きく変える場合は両方を揃える必要がある。

## レンダリングループ

`loop(now)` が毎フレーム: `rainDraw → draw → spDraw → reconDraw` を描画し、`accdc/acc1/acc2` のスロットルで `tickDecrypt`(0.07s)・各種ゲージ/ロスター/16進(0.5s)・ログ(1.6s)を更新。`requestAnimationFrame` で継続。

## 既知の落とし穴（再発防止メモ）

- **東西反転**: `project` は `sx=CX-X`、`fillGlobe` の逆投影は `X=-nx`。両方セットで初めて正しい向き。片方だけ直すと地図と塗りがズレる。
- **衛星の軌道半径**: 必ず描画時に `R*rmul`。初期化時に `R` を掛けない。
- **正射投影で統一**: 塗り(マスク逆投影)が正射のため、線も正射でないと縁がズレる。透視投影を足すなら両方に入れる。
- **変数名衝突**: レインの文字セット定数 `RCH` と再構築球の高さで衝突した過去あり（高さは `REH` に改名済み）。新規変数の命名に注意。
- **マスク塗りはベクタ塗りに戻さない**: 巨大リングで破綻する。

## 埋め込みデータの再生成（必要時のみ）

`COAST`(陸地リング) と `MASK_B64`(陸地ビットマップ) は Natural Earth 110m から生成して埋め込んでいる。解像度を上げたい等の場合のみ再生成する。

- 取得元（要ネットワーク）:
  - 陸地ポリゴン: `raw.githubusercontent.com/martynafford/natural-earth-geojson/master/110m/physical/ne_110m_land.json`
  - 海岸線(参考): 同 `ne_110m_coastline.json`
- 手順の概要:
  1. land.json の各リングを `[lon,lat]` で取り出し、1桁丸め＋近接点間引きして `const COAST=[[lon,lat,...], ...]` を生成し、データ用 `<script>` に差し替え。
  2. 同じ land.json から、緯度経度を縦方向レイ法で内外判定して 480×240 の1bitビットマップを作り、`Buffer→base64` で `MASK_B64` を生成し差し替え。`MASK_W/MASK_H` も合わせる。
  3. 反子午線(±180°)をまたぐ辺は判定からスキップ（縦レイ法）。
- ヘッドレス検証には `@napi-rs/canvas`(プリビルド) が使える。`canvas`(node-gyp) は環境によりビルド不可。

## よくある調整の早見表

| やりたいこと | 触る場所 |
|---|---|
| 地軸を傾ける | `const ROLL`（0=垂直、`-23.5*Math.PI/180`等で傾き） |
| 自転速度 | `draw()` 内 `earthSpin += dt*0.10`、`simTime += dt*0.10` |
| 衛星の数/高さ/速度 | `sats` 生成ループ（`rmul`/`speed`/`inc`） |
| 大陸の色・濃さ | `fillGlobe()` の `d[p]=..` RGBA |
| 海岸線/経線の濃さ | `draw()` の coastline / `dl()` の `globalAlpha`・色 |
| コードレインの強さ | CSS `#rain{opacity}` ＋ `rainDraw` のフェード/色/スロットル |
| デクリプト文言・速度 | `DC_TARGETS`、`tickDecrypt` の `dcProg += ...`、`accdc>0.07` |
| 再構築球の速度/細かさ/色 | `reconDraw`(`reconP += dt*0.17`, `fresh`色)、`QUADS`(`latN/lonN`) |
| 全体の配色 | `:root` トークン ＋ 各キャンバス描画のハードコード色 |
